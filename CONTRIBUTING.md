# Contributing to Digispark HID Toolkit

Thank you for considering a contribution. Every improvement helps make this project more useful for the security education community.

---

## Ways to Contribute

- Report bugs via GitHub Issues
- Suggest new features or payload examples
- Improve documentation or fix typos
- Submit new example sketches
- Translate documentation

---

## Before You Start

- Check existing issues to avoid duplicates
- For large changes, open an issue first to discuss the approach
- This project is for educational use only. Do not submit payloads designed for unauthorized access

---

## Pull Request Process

1. Fork the repository
2. Create a branch from `main`
   ```
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Test on actual hardware if possible
5. Write clear commit messages
6. Push and open a Pull Request against `main`

---

## Code Style

- Arduino sketches: follow standard Arduino formatting
- PowerShell scripts: use consistent indentation and variable naming
- Markdown: one sentence per line for easier diffs

---

## Commit Messages

Use this format:
```
type: short description

Optional longer explanation.
```

Types: `feat`, `fix`, `docs`, `refactor`, `chore`

Example: `docs: add Discord webhook setup guide`

---

## Reporting Issues

Include:
- Operating system and version
- Arduino IDE version
- Digispark board version
- Steps to reproduce
- Expected vs actual behavior

---

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating you agree to uphold these standards.
